# Forward-Proxy Allowlist — eine Domain je Zeile.
#   .domain  -> inkl. der Domain UND aller Subdomains (.npmjs.org deckt registry.npmjs.org)
#   domain   -> exakter Host (ohne fuehrenden Punkt)
# Gelesen fuer HTTP (dstdomain) UND HTTPS-SNI (ssl::server_name) -> eine Wahrheit.
# Aenderung wirkt per `docker compose exec forward-proxy squid -k reconfigure`.
#
# WICHTIG: Paketmanager folgen oft Redirects auf CDNs -> die transitiven Hosts
# muessen mit drauf, sonst bricht der Build still (siehe 04-testing.md).
# Hinweis: keine exakten Hosts eintragen, die schon von einem .domain-Eintrag
# abgedeckt sind (Squid warnt sonst ueber Redundanz).
